# Shipway API

## Auth Method

- **username** (login ID) + **password** (license key) included in every request body.
- Credentials are sent in the registration email from Shipway after signup.
- No header-based auth or Bearer tokens; credentials travel in the JSON payload.

## Base URL

| Environment | Base URL |
|---|---|
| Production | `https://shipway.in/api` |

Additional docs sites: `apidocs.shipway.com` (shipping APIs), `apidocs.shipway.in` (experience/post-purchase APIs).

## Key Endpoints

### Push Order Data

- **POST** `/pushOrderData`
- Pushes order/shipment data into Shipway for tracking and management.
- Required fields: `username`, `password`, `carrier_id` (numeric courier ID), `awb` (tracking number from the carrier), `order_id`, `first_name`, `last_name`, `email`, `phone`, `products` (text description), `company`.
- Shipway requires a pre-existing AWB from the underlying carrier; it does not generate AWBs.
- Example request:
  ```json
  {
    "username": "user_demo",
    "password": "license_key_here",
    "carrier_id": "1",
    "awb": "12971049xxxxxx",
    "order_id": "10",
    "first_name": "John",
    "last_name": "Doe",
    "email": "john@example.com",
    "phone": "9876543210",
    "products": "Widget x2",
    "company": "MyStore"
  }
  ```

### Get Order Shipment Details (Tracking)

- **POST** `/getOrderShipmentDetails`
- Required fields: `username`, `password`, `order_id`.
- Must first push data via `pushOrderData` before tracking.
- Response includes current status, scan history with timestamps, status descriptions, and locations.

### List Carriers

- **GET** `/carriers`
- Returns available carrier IDs and names. Use these IDs in `carrier_id` for pushOrderData.

### Cancellation

- No direct cancellation endpoint found in public docs.
- Cancellation must happen on the underlying carrier API.

### Serviceability

- No direct pincode/serviceability check endpoint.
- Serviceability depends on the underlying carrier.

## Request Format

All endpoints use **POST** with `Content-Type: application/json`. Auth credentials are included alongside data fields:

```json
{
  "username": "...",
  "password": "...",
  ...endpoint-specific fields...
}
```

## Credentials

- `username` — Shipway login ID
- `licenseKey` — Shipway license key (sent as `password` in API calls)
- `carrierId` — optional default carrier ID
- `companyName` — company name for order registration

## Notes

- Shipway is a tracking/notification aggregator integrating with 600+ carriers.
- AWBs must be generated by the underlying carrier before pushing to Shipway.
- Supports reverse logistics and return management.
- India-focused with some global carrier support.
